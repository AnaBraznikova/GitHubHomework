

1. git clean  отменит изменения в текущей директории
2. git status ---> git checkout -- <file>  удалит его из индекса
3. git commit --amend
4. git branch --->  git checkout input   либо   git checkout -b new_branch_name
5. если в ветках нет расхождений в изменениях, то GIT идёт по лёгкому пути упрощая себе работу, и просто переносит указатель к ветке, т.е. git merge -ff; 
если же изменения есть, то ветки не могут слиться, и тогда создастся новый коммит в активной ветке. git merge --no-ff
6. когда в ветках, в тех же файлах на той же строке, есть разница. Тогда git спросит какой файл с каким изменением оставить. И уже тогда либо удалить файл, ли исправить, чтобы всё совпадало
7.git branch -d branch_name
